## Experience Cloud

### Marketing Cloud ID サービス {#mcvid}

このリリースの ID サービスのバージョンは 1.7.0 となります。以下の機能、問題修正、その他の修正点が含まれています。

**修正点および改善点**

* 関数 ` Visitor.getInstance` で設定できるオプションのブール値のフラグとして、` idSyncAttachIframeOnWindowLoad` が追加されました。` idSyncAttachIframeOnWindowLoad= true` の場合、ID サービスは、ウィンドウの読み込み時に ID 同期 iFrame を読み込みます。デフォルトでは、ID サービスは、可能な限り迅速に iFrame を読み込みます。このフラグは、廃止される *に代わるものです。*` idSyncAttachIframeASAP`[ID サービスの getInstance 関数で使用される変数](https://marketing.adobe.com/resources/help/en_US/mcvid/?f=mcvid-variables.html)を参照してください。（MCID-122）
* ドメイン、ネイティブアプリおよびハイブリッドアプリから Web 移行への [!DNL  Marketing Cloud] ID のトラッキングをサポートする機能が追加されました。[訪問者 ID 追加ヘルパー関数](https://marketing.adobe.com/resources/help/en_US/mcvid/mcvid-appendvisitorid.html)を参照してください。（MCID-42）
* ID サービスが訪問者 [!DNL  Marketing Cloud] ID をクライアント側またはサーバー側に生成したかどうか、または ID 呼び出しがタイムアウトしたかどうかを判断する関数が visitorAPI.js コードに追加されました。[タイムアウトトラッキング関数](https://marketing.adobe.com/resources/help/en_US/mcvid/?f=mcvid-timeout-functions.html)および[クライアント側訪問者 ID 生成のトラッキング](https://marketing.adobe.com/resources/help/en_US/mcvid/?f=mcvid-client-side-id.html)を参照してください。（MCID-116）

**既知の問題**

[!DNL  Audience Manager] DIL コードと visitorAPI.js コードを同じページに使用しているお客様は、DIL 変数 ` secureDataCollection= false` を設定する必要があります。[secureDataCollection](https://marketing.adobe.com/resources/help/en_US/aam/?f=dil-secure-data-collection.html) を参照してください。

**メンテナンスリリース：2016 年 8 月 18 日**

**DIL の問題の修正、機能拡張および廃止されたコード**

DIL は、[!DNL  Audience Manager] の API データ収集コードです。7 月のリリースの DIL のバージョンは 6.5 となります。これらの機能および修正点を活用するには、最新バージョンにアップグレードします。

* DIL コードが Firefox にデプロイできなかった問題を修正しました。（AAM-24708）
* DIL iFrame にタイトル属性が追加されました。iFrame のタイトルを指定すると、目の不自由な利用者向けにスクリーンリーダーで読み上げ可能なページ情報を提供できます。（AAM-25317）
* クロスオリジンリソース共有（CORS）の DIL サポートを追加しました。[useCorsOnly](https://marketing.adobe.com/resources/help/en_US/aam/?f=dil-use-cors-only.html) を参照してください。（AAM-25319、AAM-20139）
* DIL iFrame は、` iframe.sandbox='allow-scripts allow-same-origin';` 属性をサポートします。これら 2 つのトークンのみを許可することで、セキュリティを強化できます。詳細については、この [iFrame ドキュメント](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/iframe)の属性の節を参照してください。（AAM-23335）
* DIL は、デフォルトで安全な HTTPS 呼び出しをおこなうようになりました。[!DNL  Audience Manager] DIL コードと visitorAPI.js コードを同じページに使用しているお客様は、DIL 変数 ` secureDataCollection= false` を設定する必要があります。[secureDataCollection](https://marketing.adobe.com/resources/help/en_US/aam/?f=dil-secure-data-collection.html) を参照してください。

**修正点、機能強化および廃止された機能**

* Facebook 指標データ収集の頻度およびライフタイムが増加しました。新しいデータ収集期間を次に示します。

   * すべての Facebook 指標は、投稿がおこなわれてから最初の 72 時間は毎時間収集されます。
   * 次の 72 時間は 8 時間ごとにデータ収集が継続されます。
   * 次の 60 日以降は、毎日データ収集が継続されます。

* 投稿一覧レポートの書き出しで、Twitter の地域ターゲット列を追加しました。
* 5 つの新しい Facebook の反応指標は、現在、投稿一覧レポートおよびプロパティの分析レポートで利用できます。「超いいね！」、「すごいね」、「悲しいね」、「うけるね」および「ひどいね」が含まれます。この後のリリースでは、Facebook の反応指標は、Adobe Analytics レポートでも利用できるようになります。
* [!DNL  Social Buzz]内のプライベートの Twitter 投稿で虫眼鏡をクリックすると、Twitter で元の投稿を開くのではなく、同じページに戻される問題を修正しました。
* 一部のエスカレーションされたツイートを要求できなかった問題を修正しました。
* 異なる日に生成されたソーシャルバズレポートでいくつかの投稿が欠落していた問題を修正しました。
* ログアウトしてから戻ると、統合モデレート内のフィード設定の変更が保存されていなかった問題を修正しました。
* 引用符で囲まれたツイートの URL が、想定された 24 文字を超過できていた問題を修正しました。
